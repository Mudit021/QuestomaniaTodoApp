# Contributing to Questomania

Thank you for your interest in contributing to Questomania! We welcome contributions from developers, designers, testers, and anyone who wants to help improve this project.

## Ways to Contribute

You can help in several ways:

- Report bugs or issues
- Suggest new features or improvements
- Improve documentation
- Fix code issues or add enhancements
- Help with UI/UX improvements
- Add tests and improve reliability

## Before You Start

Please make sure you have:

- a working Android development environment
- the project cloned locally
- a basic understanding of Kotlin and Android development

## How to Contribute

1. Fork the repository.
2. Create a new branch for your change.
3. Make your changes and test them locally.
4. Commit your work with a clear message.
5. Open a pull request describing what you changed and why.

## Contribution Guidelines

- Keep changes focused and easy to review.
- Follow the existing coding style and project structure.
- Write clear commit messages.
- If you add a new feature, explain its purpose and expected behavior.
- If you fix a bug, include steps to reproduce it when possible.

## Pull Request Expectations

When submitting a pull request:

- describe the problem you solved
- explain the approach you took
- mention any relevant files or modules
- include screenshots if the change affects the UI

## Code of Conduct

Please be respectful, constructive, and collaborative in all discussions and contributions. We want this project to remain welcoming to everyone.

## Questions?

If you have questions or want to discuss an idea before contributing, feel free to open an issue or start a discussion in the repository.

We appreciate your support and look forward to your contributions!
